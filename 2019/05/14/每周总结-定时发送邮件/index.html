rails定时发送邮件

  • 用到的技术：ActiveMailer、gem whenever、cronbtab、rake

  • 最初的问题是定时任务默认的环境是producution，需要改成development，可以在schedule.rb中设置 set :environment, :development 使用gem：whenever时手动修改cron list没用，因为whenever –update-crontab（或者whenever -i）时会重新将schedule.rb文件重新执行生成新的cron list, 手动改动的版本就不会被保存会被覆盖。只要决定了使用gem，就要基于gem修改了，因为gem在顶层之上。看错误日志还是可以看crontab的，linux会默认关掉log，需要手动将那行设置的注释去掉（sudo gedit /etc/rsyslog.d/50-default.conf），并且本地可能没有安装邮件服务，需要安装（sudo apt-get install postfix sudo service postfix start）。也可以在schedule.rb中设置错误日志的输出　set :output, {:error => ‘~/desktop/z.error.log’, :standard => ‘~/desktop/z.standard.log’}另外，如果你的rails框架是使用rvm或者rbenv安装的，那么cron 会找不到bundle命令，手动cron list中添加bundle路径的方式不可取，可以在schedule.rb中添加　ENV.each { |k, v| env(k, v) }　

  • 总结：查官方文档；检查错误日志；google前人经验；同样一个gem的许多功能和需要的配置在常常就一个文件里面就可以搞定，比如定时任务的schedule.rb
